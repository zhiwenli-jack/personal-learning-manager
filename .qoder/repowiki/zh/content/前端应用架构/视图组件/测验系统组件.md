## 简介
本文件面向"测验系统组件"的完整使用与开发文档，覆盖从测验创建、答题到结果展示的全流程，重点说明：
- 开始页面的配置项：学习方向、测验模式（不限时/限时）、评分方式（百分制/等级制）、题目数量等
- 答题页面的实时交互：倒计时显示、题目导航、答案保存与提交
- 结果页面的数据呈现：得分统计、错题分析、AI评语与解析，以及新增的游戏化奖励反馈
- 删除功能：测验删除、放弃测验、错题移除的完整流程
- 测验状态管理：开始、进行中、完成状态切换
- 实时答题的性能优化与用户体验设计
- 测验历史记录、重复练习与错题本联动
- **新增**：游戏化反馈系统，包括经验值获取、等级提升、成就解锁和每日任务完成提示

## 详细组件分析

### 开始页面（ExamStartView.vue）
- 配置项
  - 学习方向：从后端获取方向列表并绑定
  - 测验模式：不限时/限时（限时需设置时间限制）
  - 评分方式：百分制/等级制（A/B/C/D）
  - 题目数量：范围 5-50，默认 10
- 行为
  - 创建测验：向后端提交配置，成功后跳转至答题页
  - 历史测验：展示最近测验，支持继续答题或查看结果
  - **新增**：删除历史测验记录，删除后刷新列表
- 交互细节
  - 仅当选择了方向时启用"开始测验"按钮
  - 加载失败时弹出提示

### 答题页面（ExamTakingView.vue）
- 实时交互
  - 倒计时：限时模式下启动定时器，剩余时间小于等于 0 自动提交
  - 进度条：根据已答题数量动态计算百分比
  - 导航：底部圆点可快速跳转到任意题目
  - 答案保存：使用响应式对象 answers 按题目 ID 保存当前答案
- 题型支持
  - 单选/多选/判断：使用单选/复选框
  - 简答题：使用多行文本输入
- 提交逻辑
  - 若存在未答题，弹窗确认后才允许提交
  - 将答案序列化为后端期望格式并提交
  - 提交完成后跳转至结果页
- **新增**：放弃测验功能
  - 点击"放弃测验"按钮，弹窗确认后删除未完成的测验记录并返回开始页

## 删除功能

### 删除测验流程
- **开始页面删除历史测验**
  - 在历史测验列表中，每个测验卡片右侧提供"删除"按钮
  - 点击后弹出确认对话框，确认后调用 DELETE /exams/{id} 接口
  - 删除成功后自动刷新测验列表
- **结果页面删除已完成测验**
  - 在结果页面底部提供"删除测验"按钮
  - 点击后弹出确认对话框，确认后调用 DELETE /exams/{id} 接口
  - 删除成功后跳转到首页
- **答题页面放弃测验**
  - 在答题页面顶部提供"放弃测验"按钮
  - 点击后弹出确认对话框，确认后调用 DELETE /exams/{id} 接口
  - 删除成功后跳转到开始页

## 游戏化反馈系统

### 经验值系统
- **经验值计算规则**
  - 基础经验值：50分
  - 分数加成：根据测验分数计算，每1分获得0.5分经验值
  - 连续登录加成：根据连续学习天数计算，3天20分、7天50分、14天100分、30天200分
- **经验值用途**
  - 等级提升：累积经验值达到升级阈值时自动升级
  - 成就解锁：部分成就解锁时获得额外经验值奖励
  - 任务完成：每日任务完成后获得经验值奖励

### 等级与头衔系统
- **等级阈值**
  - 通过`LEVEL_THRESHOLDS`配置，每个等级需要累积的经验值阈值不同
  - 最大等级：`MAX_LEVEL`限制最高等级
- **头衔系统**
  - 根据等级自动分配头衔，如"见习探险家"、"初级学者"、"高级专家"等
  - 升级时显示从旧头衔到新头衔的变更

### 成就系统
- **成就分类**
  - 测验类：首次测验、测验大师、满分学霸等
  - 错题类：知错能改、错题猎人等
  - 资料类：知识收集者等
  - 探索类：多面手、全知之眼等
  - 连续学习类：七日修行、坚持不懈等
- **稀有度系统**
  - 普通（50经验）、稀有（100经验）、史诗（200经验）、传说（200经验）
- **解锁条件**
  - 基于用户统计数据自动检测，如测验次数、正确率、掌握错题数等

### 每日任务系统
- **任务池**
  - 包含多种任务类型：每日挑战、追求完美、错题复习、知识积累、题海战术
- **任务生成**
  - 每日从任务池中随机抽取固定数量的任务
  - 任务目标和经验值奖励各不相同
- **任务进度**
  - 自动跟踪用户完成进度，完成后自动发放经验值奖励

### 测验完成后的奖励处理
- **奖励流程**
  1. 计算经验值（基础+分数+连续登录加成）
  2. 检测是否升级，更新用户等级和头衔
  3. 更新方向探索进度
  4. 更新每日任务进度
  5. 检测成就解锁条件
  6. 返回奖励数据给前端
- **奖励数据结构**
  - `exp_gained`: 获得的经验值
  - `level_up`: 是否升级
  - `old_level/new_level`: 旧等级和新等级
  - `new_title`: 新头衔
  - `achievements_unlocked`: 新解锁成就列表
  - `tasks_completed`: 已完成任务列表

```mermaid
flowchart TD
ExamComplete["测验完成"] --> CalcEXP["计算经验值"]
CalcEXP --> AwardEXP["发放经验值"]
AwardEXP --> CheckLevel["检测升级"]
CheckLevel --> |升级| UpdateLevel["更新等级和头衔"]
CheckLevel --> |未升级| CheckAchieve["检测成就"]
UpdateLevel --> CheckAchieve
CheckAchieve --> UpdateProgress["更新方向进度"]
UpdateProgress --> UpdateDaily["更新每日任务"]
UpdateDaily --> UnlockAchieve["解锁成就"]
UnlockAchieve --> ReturnData["返回奖励数据"]
ReturnData --> Frontend["前端显示奖励反馈"]
```

## 故障排查指南
- 开始页无法创建测验
  - 检查是否选择了学习方向
  - 查看网络请求返回的错误信息
- 答题页无法加载或提交失败
  - 确认测验 ID 是否有效
  - 检查后端日志与数据库连接
- 结果页为空或报"测验尚未完成"
  - 确认测验状态是否为已完成
  - 检查提交接口是否成功写入答案与评分
  - **新增**：检查sessionStorage中是否存在gamification数据
- **新增**：删除功能相关问题
  - 确认删除按钮是否正确绑定事件
  - 检查确认对话框是否正常显示
  - 验证后端DELETE接口是否正常工作
  - 查看浏览器控制台是否有JavaScript错误
- **新增**：游戏化功能相关问题
  - 检查游戏化API是否正常工作
  - 确认用户档案是否存在且正常
  - 验证成就解锁条件是否满足
  - 查看游戏化服务层的日志输出
- 错题本无数据
  - 确认测验中是否有错误答案被记录
  - 使用过滤条件缩小范围定位问题
  - **新增**：检查删除错题功能是否正常工作
