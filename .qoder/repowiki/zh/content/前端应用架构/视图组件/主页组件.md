## 简介
本文件面向"主页组件"的完整技术文档，围绕前端主页视图进行深入剖析，涵盖整体布局与设计、英雄区域的视觉与动画、快速开始卡片的导航与用户引导、学习方向卡片的渲染与交互、模态框组件的表单与状态管理、响应式设计与动画效果、API 调用模式与错误处理、性能优化策略，以及组件复用与状态管理的最佳实践。

**更新** 本次更新重点反映了新增的探索者仪表板功能，包括用户档案、经验进度条和每日任务列表的集成，以及相关的后端游戏化服务支持。

## 项目结构
- 前端采用 Vue 3 单页应用，使用组合式 API（script setup）组织页面逻辑。
- 主页位于 views/HomeView.vue，作为应用的入口视图之一，负责展示学习路径概览、统计数据、学习方向卡片、新增弹窗，以及新增的探索者仪表板（用户档案、经验进度条、每日任务列表）。
- 全局样式集中于 style.css，定义主题变量、渐变、阴影、动画与网格布局等。
- 导航由 App.vue 的导航栏与路由配置共同构成，路由在 router/index.js 中定义，按需懒加载各视图组件。
- API 层封装于 api/index.js，统一前缀与超时配置，并暴露方向、资料、题目、测验、错题、**游戏化**等模块化接口。
- 后端游戏化服务位于 backend/app/services/gamification_service.py，提供经验值系统、成就系统、每日任务等功能。

## 核心组件
- 英雄区域（Hero Section）
  - 渐变背景与动态背景圆环：通过伪元素与旋转动画实现。
  - 浮动形状动画：三个圆形形状以不同延迟与幅度浮动旋转。
  - 统计信息展示：学习方向数量、学习资料数量、生成题目数量。
- **探索者仪表板（Explorer Dashboard）**
  - **用户档案卡片**：显示用户名、等级、经验、称号、连续天数等个人信息。
  - **经验进度条**：实时显示当前经验到下一级的经验进度，包含百分比计算和动画效果。
  - **每日任务列表**：展示今日任务，包含任务名称、描述、进度条、奖励经验和完成状态。
- 快速开始卡片（Quick Actions）
  - 三卡片布局，分别链接至资料上传、开始测验、错题本。
  - 卡片悬停效果、图标放大、箭头滑入等动画。
- 学习方向卡片（Directions Section）
  - 列表渲染，带入场动画；支持删除与查看详情。
  - 新增方向弹窗，包含表单校验与提交。
- 模态框组件（Modal）
  - 基于 Vue 过渡系统的弹窗，包含遮罩层、缩放动画与表单控件。
- 响应式设计
  - 在移动端调整布局、字体大小与卡片尺寸，保证可用性。

### 探索者仪表板（Explorer Dashboard）

**更新** 新增探索者仪表板功能，包含用户档案、经验进度条和每日任务列表。

#### 用户档案卡片（Explorer Profile）
- 设计要点
  - **等级徽章**：显示当前等级，使用渐变背景和阴影效果。
  - **个人信息**：用户名和称号显示，体现角色成长。
  - **连续天数**：火焰图标显示连续学习天数，使用橙色强调。
  - **统计信息**：总经验、等级、连续天数的数值显示。
- 数据绑定与状态管理
  - 通过 gamificationApi.getProfile() 获取用户档案数据。
  - 包含用户名、等级、总经验、下一级所需经验、称号、连续天数等字段。
  - 支持跳转到探险地图视图的功能。
- 交互效果
  - 卡片使用渐变背景和模糊滤镜，增强视觉层次。
  - 按钮采用块级显示，提供统一的导航入口。

#### 经验进度条（Experience Progress Bar）
- 计算逻辑
  - **百分比计算**：通过 expPercentCalc 计算当前等级的经验进度百分比。
  - **区间计算**：基于当前经验、下一级所需经验和当前等级起始经验计算。
  - **边界处理**：当达到最高等级时显示100%进度。
- 视觉效果
  - **渐变填充**：使用渐变色填充进度条，增强视觉吸引力。
  - **动画过渡**：进度变化时使用缓动函数，提供流畅的视觉体验。
  - **阴影效果**：进度条具有发光阴影，提升立体感。
- 文本显示
  - 显示当前经验/下一级所需经验的格式化文本。
  - 字体使用渐变色，突出重要信息。

#### 每日任务列表（Daily Tasks）
- 渲染机制
  - 使用 v-for 遍历任务数组，支持任务完成状态的视觉区分。
  - **空状态**：当任务列表为空时显示加载提示。
  - **完成状态**：已完成任务使用绿色主题和勾选图标。
- 任务信息
  - **任务名称**：显示任务标题和描述。
  - **进度条**：显示任务完成进度，支持实时更新。
  - **奖励经验**：显示完成任务获得的经验奖励。
- 交互设计
  - 任务项采用卡片式设计，带有边框和背景色。
  - 完成状态的任务具有特殊的绿色主题样式。
  - 进度条使用渐变色填充，提供直观的完成度指示。

## 性能考虑
- 并发加载
  - 使用 Promise.all 同时获取方向、统计数据、**用户档案**和**每日任务**，缩短首屏等待时间。
  - **优化**：探索者仪表板的数据加载与主数据并行执行，提升整体响应速度。
- 懒加载与按需路由
  - 路由器对视图组件使用动态导入，降低初始包体积。
- 动画与渲染
  - 控制动画复杂度与数量，避免在低端设备上造成掉帧。
  - **经验进度条**：使用缓动函数和合理的动画时长，平衡视觉效果与性能。
- 网络超时
  - axios 超时设为较长时长，适配 AI 处理耗时场景，必要时可增加重试与进度提示。
- 最佳实践
  - 对高频更新的状态使用浅层响应式，避免深层嵌套导致的性能问题。
  - 在移动端优先使用 transform 与 opacity 动画，减少重排重绘。
  - **探索者仪表板**：经验进度计算使用计算属性缓存，避免重复计算。

## 故障排除指南
- 加载失败
  - 现象：统计或方向列表为空或报错。
  - 排查：检查网络请求与后端接口状态，确认 axios 配置与路由正确。
  - **探索者仪表板**：检查 gamification API 是否正常工作，确认用户档案和每日任务接口。
- 新增失败
  - 现象：弹窗提交后无反应或弹出错误提示。
  - 排查：确认必填字段、后端返回的错误信息，检查 directionsApi.create 的调用链。
- 删除失败
  - 现象：删除确认后仍存在或提示失败。
  - 排查：确认确认对话框逻辑与 directionsApi.delete 的调用，检查后端权限与数据约束。
- 动画异常
  - 现象：卡片入场或模态框动画不生效。
  - 排查：检查 CSS 变量与动画类是否正确引入，确认浏览器兼容性。
  - **经验进度条**：检查 expPercentCalc 计算逻辑，确认数据格式正确。
- **探索者仪表板特定问题**
  - **进度条不更新**：检查 expPercentCalc 计算逻辑，确认 profile 数据完整性。
  - **任务列表为空**：确认 gamificationApi.getDailyTasks 是否正确返回数据。
  - **用户档案显示异常**：检查 gamificationApi.getProfile 的数据结构和字段映射。
